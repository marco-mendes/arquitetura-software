### 3.3 — Refatoração crítica

O código abaixo tenta implementar Pipes and Filters, mas viola os princípios do padrão. Identifique todos os problemas, classifique-os e proponha a refatoração.

```python
import sqlite3

class PipelineProcessamento:
    resultado_global = []  # acumulador compartilhado entre instâncias

    def processar_tudo(self, registros: list[dict]) -> list[dict]:
        # Passo 1: valida
        validados = []
        for r in registros:
            if r.get("valor", 0) > 0:
                validados.append(r)

        # Passo 2: enriquece — acessa banco diretamente
        conn = sqlite3.connect("sistema.db")
        enriquecidos = []
        for r in validados:
            cursor = conn.cursor()
            cursor.execute("SELECT nome FROM clientes WHERE id = ?", (r["cliente_id"],))
            nome = cursor.fetchone()[0]
            r["nome_cliente"] = nome
            enriquecidos.append(r)
        conn.close()

        # Passo 3: calcula média — usa estado acumulado
        for r in enriquecidos:
            self.resultado_global.append(r["valor"])
            r["media_historica"] = sum(self.resultado_global) / len(self.resultado_global)

        return enriquecidos
```

Para cada problema encontrado:
- Identifique qual princípio do Pipes and Filters está sendo violado
- Classifique: **crítico** (inviabiliza testes ou paralelismo) ou **importante** (viola boas práticas)
- Proponha como refatorar usando o framework `Filtro` / `Pipeline` de *1.3*
