# Bibliografia

A bibliografia combina fontes primárias, que definem técnicas e padrões, com livros-base que apoiam análise e aplicação. Os módulos indicam as leituras diretamente relacionadas a cada encontro.

## Fontes primárias

- FIELDING, Roy Thomas. *Architectural Styles and the Design of Network-based Software Architectures*. Tese de doutorado, University of California, Irvine, 2000.
- BASS, Len; CLEMENTS, Paul; KAZMAN, Rick. *Software Architecture in Practice*. 4. ed. Addison-Wesley, 2021.
- CLEMENTS, Paul et al. *Documenting Software Architectures: Views and Beyond*. 2. ed. Addison-Wesley, 2010.
- NYGARD, Michael. “Documenting Architecture Decisions”. 2011.
- OPENAPI INITIATIVE. *OpenAPI Specification*.
- CLOUD NATIVE COMPUTING FOUNDATION. *OpenTelemetry Specification*.
- CLOUD NATIVE COMPUTING FOUNDATION. *Kubernetes Documentation*.
- AMAZON WEB SERVICES. *The Amazon Builders' Library*.

## Livros-base

- RICHARDS, Mark; FORD, Neal. *Software Architecture: The Hard Parts*. O’Reilly, 2021.
- NEWMAN, Sam. *Building Microservices*. 2. ed. O’Reilly, 2021.
- KLEPPMANN, Martin. *Designing Data-Intensive Applications*. O’Reilly, 2017.

- HUMBLE, Jez; FARLEY, David. *Continuous Delivery*. Addison-Wesley, 2010.

## Como ler

Use especificações para confirmar semântica e limites de uma tecnologia. Use livros para comparar forças e consequências. Estudos de fornecedores podem oferecer evidência operacional, mas devem ser lidos considerando o contexto em que a experiência ocorreu.
