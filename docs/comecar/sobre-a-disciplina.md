# Sobre a disciplina

Arquitetura de Software é uma disciplina de pós-graduação com 24 horas, organizadas em seis encontros de quatro horas. O curso atende profissionais que já desenvolvem software em .NET, Java ou Python e desejam tornar decisões estruturais mais explícitas, comparáveis e verificáveis.

## Pré-requisitos

Espera-se familiaridade com programação, versionamento de código, linha de comando e conceitos básicos de aplicações web. Experiência prévia como arquiteto não é necessária. Python é a trilha executável de referência; comparações com Java e .NET apoiam a transferência para outros ambientes.

## Os seis encontros

1. **Estilos arquiteturais:** reconhecer componentes, conectores e restrições de estilos recorrentes.
2. **APIs:** definir contratos e escolher mecanismos de integração síncrona.
3. **Serviços:** delimitar responsabilidades, dados e colaboração distribuída.
4. **Governança:** aplicar políticas e observar o comportamento de serviços.
5. **Eventos:** projetar publicação, consumo e tratamento de repetição e falha.
6. **Nuvem:** relacionar implantação e operação a atributos de qualidade.

## Método de aprendizagem

Cada encontro parte de conceitos e vocabulário comum, avança para exemplos arquiteturais e aplica as decisões ao mesmo caso hospitalar. A oficina torna evidências observáveis; os exercícios organizados pela Taxonomia de Bloom permitem escolher a profundidade adequada; o projeto integrador registra a evolução em artefatos pequenos e conectados.

As decisões são discutidas por contexto, forças, alternativas e consequências. Ferramentas aparecem para tornar um fenômeno visível, nunca como substitutas do raciocínio arquitetural.
