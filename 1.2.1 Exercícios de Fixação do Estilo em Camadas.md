### 3.3 — Refatoração crítica

O código abaixo é um Controller de uma API de pedidos. Identifique todos os problemas arquiteturais, classifique-os por gravidade e proponha a refatoração.

```python
import sqlite3
import smtplib
from dataclasses import dataclass

@dataclass
class PedidoController:
    def criar_pedido(self, cliente_id: int, itens: list[dict], cupom: str) -> dict:
        # Valida diretamente no banco
        conn = sqlite3.connect("loja.db")
        cursor = conn.cursor()
        cursor.execute("SELECT credito FROM clientes WHERE id = ?", (cliente_id,))
        credito = cursor.fetchone()[0]

        # Calcula desconto aqui mesmo
        desconto = 0.0
        if cupom == "PROMO10":
            desconto = sum(i["preco"] for i in itens) * 0.10
        elif cupom == "FIDELIDADE20":
            desconto = sum(i["preco"] for i in itens) * 0.20

        # Verifica se tem crédito
        total = sum(i["preco"] for i in itens) - desconto
        if credito < total:
            return {"erro": "Crédito insuficiente"}

        # Insere diretamente
        cursor.execute(
            "INSERT INTO pedidos (cliente_id, total) VALUES (?, ?)",
            (cliente_id, total)
        )
        conn.commit()
        conn.close()

        # Envia e-mail diretamente do Controller
        with smtplib.SMTP("smtp.loja.com") as smtp:
            smtp.sendmail("no-reply@loja.com", f"cliente{cliente_id}@mail.com",
                          f"Pedido criado! Total: R${total:.2f}")

        return {"status": "criado", "total": total}
```

Para cada problema encontrado:
- Identifique qual princípio ou responsabilidade está sendo violado
- Classifique: **crítico** (impede testes, cria acoplamento permanente) ou **importante** (viola boas práticas mas pode ser corrigido incrementalmente)
- Proponha como refatorar sem reescrever tudo de uma vez
